# 健身 App 产品功能规格（海外市场）

**目标市场**：海外（英语区优先，订阅制）  
**核心方向**：场景化短时训练 + 体态康复  
**文档用途**：MVP 与迭代的功能列表与优先级

---

## 一、场景化短时训练（Scenario-based Short Workouts）

### 1.1 功能模块总览

| 模块 | 说明 | 优先级 |
|------|------|--------|
| 场景库 | 按时间+地点定义训练场景 | P0 |
| 训练包 | 每个场景下的具体动作与时长 | P0 |
| 计时与引导 | 单次训练的计时、提示、完成记录 | P0 |
| 离线与轻量 | 无网可用、包体小 | P0 |
| 每日推荐 | 根据时间/场景推荐今日训练 | P1 |
| 进度与统计 | 本周完成次数、总时长 | P1 |
| 自定义场景 | 用户自建「我的场景」 | P2 |

---

### 1.2 场景库（P0）

**功能描述**：预置「时间 + 场景」维度，用户选场景即进入对应短训。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 1.1 | 场景分类入口 | 首页展示场景卡片：By Time（早晨/午休/下班/睡前）、By Place（办公室/家里/户外/旅行） | P0 |
| 1.2 | 预置场景列表 | 至少 8 个场景，例如：Morning 5min, Lunch Break 7min, Desk Stretch 5min, Before Bed 7min, Travel 5min, Living Room 10min, Outdoor 5min, Wake-up 3min | P0 |
| 1.3 | 场景详情页 | 展示该场景下的训练包名称、总时长、动作数量、难度标签 | P0 |
| 1.4 | 多语言 | 英文为首发，文案与 UI 支持后续扩展（i18n 结构） | P0 |

**验收标准**：用户能通过「场景」找到并进入一个具体训练包，无需登录即可体验 1 个免费场景。

---

### 1.3 训练包与内容（P0）

**功能描述**：每个场景对应 1 个或多个「训练包」，包内为固定顺序的动作 + 时长。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 2.1 | 训练包数据结构 | 包含：packId, sceneId, name, totalMinutes, difficulty, exercises[]（exerciseId, name, durationSeconds, restSeconds, tip, image/videoUrl） | P0 |
| 2.2 | 动作类型 | 支持：静态拉伸、计时保持、重复次数（如 10 次）。MVP 可先做「按秒计时」一种 | P0 |
| 2.3 | 内容数量 | MVP 至少 4 个场景各有 1 个训练包；每个包 5–8 个动作，总时长 3–10 分钟 | P0 |
| 2.4 | 媒体资源 | 每个动作至少 1 张示意图或短 GIF；可选短视频（后续迭代） | P0 |
| 2.5 | 无器械标注 | 每个训练包标注是否需器械（MVP 全部为 No Equipment） | P0 |

**验收标准**：每个训练包可完整播放/引导一轮，时长与顺序正确。

---

### 1.4 计时与引导（P0）

**功能描述**：用户进入某个训练包后，按动作顺序进行计时与切换，并有简单引导。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 3.1 | 动作页 | 当前动作名称、示意图、倒计时/正计时、可选语音或震动提示 | P0 |
| 3.2 | 自动切换 | 当前动作结束后自动进入下一动作或休息；最后一动结束后进入完成页 | P0 |
| 3.3 | 暂停/继续 | 支持暂停、继续；可选「跳过当前动作」（计入完成度） | P0 |
| 3.4 | 完成页 | 显示本次总时长、完成动作数、简短鼓励文案；按钮「再来一次」「返回场景」 | P0 |
| 3.5 | 后台/锁屏 | 计时在 App 进入后台或锁屏时继续（或提示用户保持前台），不强制锁屏常亮 | P1 |

**验收标准**：从进入训练到完成，无需额外点击即可走完一轮，计时准确。

---

### 1.5 离线与轻量（P0）

**功能描述**：核心训练流程不依赖网络，安装包与资源尽量小。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 4.1 | 离线可用 | 场景列表、训练包、动作与图片在首次加载或发版时打包/预缓存，无网可进入训练 | P0 |
| 4.2 | 资源策略 | 图片压缩、可选「仅下载当前使用场景」；MVP 可先全部打包进 App | P0 |
| 4.3 | 包体目标 | 安装包 < 50MB（MVP 可放宽到 80MB） | P1 |

---

### 1.6 每日推荐与发现（P1）

**功能描述**：根据当前时间或用户选择，推荐「今日可做的场景/训练包」。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 5.1 | 首页推荐位 | 如「Right now: Desk Stretch 5min」或「Suggested for evening: Before Bed 7min」 | P1 |
| 5.2 | 简单规则 | 按本地时间匹配（如 12:00–14:00 推午休场景）；无账号时仅用时间 | P1 |
| 5.3 | 历史偏好 | 若用户有完成记录，可优先推荐常做场景（需账号，P2） | P2 |

---

### 1.7 进度与统计（P1）

**功能描述**：记录用户完成情况并做轻量展示，提升留存。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 6.1 | 完成记录 | 每次训练结束写入：packId, date, durationSeconds, completed（是否全部做完） | P1 |
| 6.2 | 本周统计 | 本周完成次数、总分钟数；可在首页或「Progress」页展示 | P1 |
| 6.3 | 简单 streak | 连续 N 天有完成记录即显示「N day streak」（可选，P2） | P2 |

**依赖**：需要用户标识（匿名 ID 或账号），本地存储 + 可选云端同步。

---

### 1.8 自定义场景（P2）

**功能描述**：用户自建「我的场景」，从已有动作库中选动作、排顺序、设时长。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 7.1 | 动作库 | 所有预置动作可被挑选（列表+搜索） | P2 |
| 7.2 | 创建流程 | 选动作 → 排顺序 → 设每动时长/休息 → 命名保存 | P2 |
| 7.3 | 我的场景列表 | 展示、编辑、删除自定义包；与预置场景一起出现在「我的」或场景列表 | P2 |

## 二、体态康复（Posture & Recovery）

### 2.1 功能模块总览

| 模块 | 说明 | 优先级 |
|------|------|--------|
| 问题/部位选择 | 用户选择不适部位或目标（颈、肩、腰、膝等） | P0 |
| 康复计划 | 针对该部位的每日/每周动作序列 | P0 |
| 计划执行与引导 | 按日推送、计时引导、完成打卡 | P0 |
| 进度与提醒 | 每日提醒、完成记录、简单效果反馈 | P1 |
| 进阶与周期 | 根据完成情况推荐进阶或维持计划 | P2 |

---

### 2.2 问题/部位选择（P0）

**功能描述**：用户通过简单选择「哪里不舒服」或「想改善什么」进入对应康复流。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 1.1 | 部位入口 | 选项：Neck & Upper Back, Shoulders, Lower Back, Hips, Knees, Wrists（MVP 先做 2–3 个，如 Lower Back, Neck, Shoulders） | P0 |
| 1.2 | 简短问卷（可选） | 1–3 题：频率（偶尔/经常/每天）、是否确诊伤病（有/无）、目标（缓解不适/预防/术后恢复）。用于后续推荐强度，MVP 可省略 | P1 |
| 1.3 | 免责声明 | 明确「非医疗建议，严重不适请就医」；首次进入该模块必须勾选或点击同意 | P0 |

**验收标准**：用户选择部位后能进入该部位的「默认康复计划」或计划列表。

---

### 2.3 康复计划与内容（P0）

**功能描述**：每个部位有 1 个默认计划（或 2 个：轻度/进阶），计划由「每日动作序列」组成。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 2.1 | 计划结构 | planId, bodyPart, name, level, daysPerWeek, totalWeeks, dailyRoutines[]（routineId, dayIndex, exercises[]） | P0 |
| 2.2 | 每日 routine | 一天 = 一组动作（顺序+时长/次数），总时长 5–15 分钟 | P0 |
| 2.3 | 动作内容 | 与场景化训练共用动作数据结构；康复动作需标注 bodyPart、contraindications（禁忌，文案） | P0 |
| 2.4 | MVP 计划数量 | 至少 2 个部位 × 1 个计划，每计划 7 天不同 routine（或 3–4 个 routine 轮换） | P0 |
| 2.5 | 动作说明 | 每个动作有「为什么做」「注意要点」简短文案，增强信任感 | P0 |

**验收标准**：用户能为「下背」或「颈」看到并进入一个多日计划，每天有明确动作列表。

---

### 2.4 计划执行与引导（P0）

**功能描述**：用户按「第几天」执行当日 routine，体验与场景化训练一致：计时、切换、完成。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 3.1 | 计划首页 | 显示当前计划名、部位、今日是第几天、今日 routine 预览（动作数、总时长） | P0 |
| 3.2 | 开始今日训练 | 进入与「场景化训练」相同的计时引导流程（可复用同一套播放器/计时组件） | P0 |
| 3.3 | 完成今日 | 完成后标记「Day N 已完成」，可显示「明天是 Day N+1 的…」 | P0 |
| 3.4 | 补做/重做 | 允许用户补做昨天或重做某天（记录以「完成日」为准，不强制连续日期） | P1 |

**验收标准**：用户能连续多天执行同一计划并看到进度推进。

---

### 2.5 进度与提醒（P1）

**功能描述**：让用户看到「坚持了几天」并适度提醒，提高依从性。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 4.1 | 计划进度 | 计划内「已完成天数 / 总天数」、本周完成次数 | P1 |
| 4.2 | 推送/本地提醒 | 每日一次可配置时间提醒（需申请通知权限）；文案如「Time for your 7-min lower back routine」 | P1 |
| 4.3 | 简单反馈（可选） | 完成当日后弹窗：「今天做完感觉如何？」1–5 星或「更好/一样/更差」；用于后续优化计划，MVP 可只记录不展示 | P2 |

---

### 2.6 进阶与周期（P2）

**功能描述**：第一轮计划完成后，引导用户进入「维持」或「进阶」计划。

**具体功能列表**：

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 5.1 | 计划完成页 | 显示「You completed the 2-week program」+ 简短总结；CTA「Start maintenance」或「Try advanced」 | P2 |
| 5.2 | 维持计划 | 同一部位提供「每周 2–3 次」的维持用 routine，从原计划中选日或新 routine | P2 |
| 5.3 | 进阶计划 | 同一部位提供更强/更长时间的计划，标注「After completing basic」 | P2 |

---

## 三、通用与基础功能（跨两个方向）

### 3.1 账户与订阅（P0 用于变现）

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 1 | 匿名使用 | 不登录可试用：1 个场景 + 1 个康复部位的部分内容（如前 3 天） | P0 |
| 2 | 账号 | 邮箱或 Apple/Google 登录；登录后同步进度、多设备 | P0 |
| 3 | 订阅 | 月/年订阅，解锁全部场景 + 全部康复计划；价格策略另文档 | P0 |
| 4 | 恢复购买 | 支持 Restore Purchase，符合 Store 规范 | P0 |

### 3.2 数据与隐私（P0）

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 1 | 隐私政策 | 说明收集的数据（账号、完成记录、设备信息）、用途、不卖给第三方 | P0 |
| 2 | 健康数据 | 若未来接入 HealthKit，单独说明并请求权限；MVP 可不接入 | P1 |

### 3.3 导航与信息架构（P0）

| # | 功能点 | 说明 | 优先级 |
|---|--------|------|--------|
| 1 | 底部 Tab | 例如：Home / Scenarios / Recovery / Progress / Settings | P0 |
| 2 | Home | 今日推荐（场景 + 康复今日任务）、最近使用、统计摘要 | P1 |
| 3 | Settings | 语言、单位、通知开关、订阅管理、隐私政策、免责声明 | P0 |

## 四、优先级汇总与 MVP 范围建议

### 4.1 必须 MVP 交付（P0）

- **场景化**：场景库（至少 4 个场景）、训练包与内容、计时与引导、离线可用、免责与多语言准备。
- **体态康复**：2 个部位（如 Lower Back + Neck）、各 1 个多日计划、计划执行与引导、免责声明。
- **通用**：匿名试用 + 账号 + 订阅 + 恢复购买、隐私政策、基础 Settings 与导航。

### 4.2 第一轮迭代（P1）

- 场景：每日推荐、本周进度与统计、完成记录与简单展示。
- 康复：计划进度展示、每日提醒、补做/重做。
- 通用：Home 聚合、通知权限与提醒。

### 4.3 后续迭代（P2）

- 场景：自定义场景、streak、历史偏好推荐。
- 康复：完成反馈、维持/进阶计划、更多部位。
- 通用：HealthKit 等生态对接（可选）。

---

## 五、技术实现提示（供开发参考）

- **场景 + 康复** 共用：动作数据模型、计时/引导 UI 组件、完成记录与统计逻辑。
- **内容**：用 JSON 或 CMS 管理场景/计划/动作，便于非开发人员增改内容。
- **i18n**：从第一天就做 key 化文案，便于后续加语言。
- **订阅**：RevenueCat 或原生 StoreKit/Google Billing，注意试用与恢复购买。

如需把某一块再拆成「接口设计」或「前端页面清单」，可以在此基础上继续细化。
